### 서블릿(WAS)이 지원해주는 기능들

```
- 서버 TCP/IP 대기, 소켓연결
- HTTP 요청 메시지를 파싱해서 읽기
- POST 방식, /save URL 인지
- Content Type 확인
- HTTP 메시지 바디 내용 파싱
- 저장 프로세스 실행

(비즈니스 로직 작성)

- HTTP 응답 메시지 생성 시작
    - HTTP 시작 라인 생성
    - Header 생성
    - 메시지 바디에 HTML 생성에서 입력
- TCP/IP에 응답 전달, 소켓 종료
```

- 톰켓 처럼 서블릿을 지원하는 WAS를 `서블릿 컨테이너`라고 한다.
- 서블릿 객체는 `싱글톤`으로 관리한다.
- 사실상 서블릿을 사용하는 이유는 동시 요청 처리를 위한 멀티 쓰레드 지원이라고 생각한다.
- MVC1(김영한)
